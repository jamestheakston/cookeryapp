API Documentation

Overview

The Cookery OAuth 2.1 server allows third-party applications to authenticate users and access their data securely. This guide will help you integrate your application with our OAuth flow.

OAuth 2.1 Authorization Code Flow with PKCE

Our OAuth implementation follows the OAuth 2.1 specification with PKCE (Proof Key for Code Exchange) for enhanced security.

Step 1: Generate PKCE Code Verifier and Challenge

// Generate code verifier (43-128 characters)
const codeVerifier = generateRandomString(128);

// Generate code challenge (SHA-256 hash, base64url encoded)
const codeChallenge = await sha256(codeVerifier);

Step 2: Redirect User to Authorization Endpoint

const authUrl = `https://ojvigxnwweixjhugekmm.supabase.co/auth/v1/oauth/authorize?` +
  `client_id=${YOUR_CLIENT_ID}&` +
  `redirect_uri=${encodeURIComponent(REDIRECT_URI)}&` +
  `response_type=code&` +
  `code_challenge=${codeChallenge}&` +
  `code_challenge_method=S256&` +
  `scope=${encodeURIComponent('openid email profile')}`;

window.location.href = authUrl;

Step 3: Exchange Authorization Code for Tokens

const response = await fetch('https://ojvigxnwweixjhugekmm.supabase.co/auth/v1/oauth/token', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/x-www-form-urlencoded',
  },
  body: new URLSearchParams({
    grant_type: 'authorization_code',
    code: authorizationCode,
    redirect_uri: REDIRECT_URI,
    client_id: YOUR_CLIENT_ID,
    code_verifier: codeVerifier
  })
});

const tokens = await response.json();

Available Scopes

  • openid - Required for OpenID Connect
  • email - Access user's email address
  • profile - Access user's profile information
  • phone - Access user's phone number
  • read:recipes - Read user's recipes
  • write:recipes - Write/create user's recipes

  "id_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Using Access Tokens

Include the access token in the Authorization header when making API requests:

const response = await fetch('https://api.cookery.com/v1/recipes', {
  headers: {
    'Authorization': `Bearer ${accessToken}`
  }
});

Client Types

  • Public Clients - Mobile apps, single-page apps. No client secret required. Uses PKCE for security.
  • Confidential Clients - Web servers with backend. Requires client secret for token endpoint authentication.
